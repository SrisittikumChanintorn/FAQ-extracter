1. Preparing Your Data

The system accepts data in three formats: Excel (.xlsx), CSV (.csv), and JSON (.json). Before uploading, prepare your file with two columns containing the customer's question and the admin/agent's answer.

Column Mapping

When you upload a file, a Data Mapping screen will appear. You will need to select which column in your file represents the Question, and which represents the Answer. The system will then automatically rename them for processing.

2. System Workflow

  • Upload Data
    Go to the Upload Data tab. Drag and drop your prepared file. Once selected, you'll be prompted to map your columns. After saving the mapping, click "Go to Process & Analyze".
  • Data Manipulation
    Before running the analysis, you can go to the Data Manipulation tab. Here, you can view up to 500 rows of your raw uploaded data in a spreadsheet-like view and make direct text edits to fix typos or remove bad data. Click "Save Changes" when done.
  • Process & Analyze
    Configure the AI settings (like duplicate strictness and minimum questions per group) and click Start Analysis. The AI will clean the data, extract intent, cluster similar questions, and generate perfect FAQ pairs. Wait for the live log to reach 100%.
  • Explore the FAQ Library
    Once finished, the FAQ Library tab shows all generated FAQs. You can filter them by "Topic Groups". Click any question to expand and read the AI-summarized answer.
  • Smart Search
    Instead of exact keyword matching, the Smart Search uses AI to understand the meaning of your search. Even if a customer asks "I can't log in", it will find the FAQ for "How to reset password".
  • Data Management
    If some FAQs are wrong or irrelevant, go to Data Management. Select the bad FAQs using the checkboxes and either delete them, or click Edit Group to reassign them to a different topic cluster. Hovering over a cluster in the edit menu will show a quick preview of other questions in that group. The system will automatically update the charts, clusters, and search index without needing to re-run the pipeline.
  • 3D Visualization
    The 3D Visualization tab plots your FAQs in a 3D semantic space. Points that are close together mean the same thing. Click a Topic Group on the right to see its questions highlighted in red.

3. Settings Explained

  • Duplicate Removal Strictness (0.70 - 0.99):
    Determines how similar two questions must be to be considered a duplicate. Very high numbers (0.95+) mean only exact matches are grouped. Lower numbers (0.80) group loosely related questions. Default is 0.92.
  • Minimum Questions per Topic Group (2 - 15):
    How many times a question needs to be asked before it becomes a "Group". If set to 3, any question asked fewer than 3 times is considered "Noise" and ignored. For small files, use 2. For massive datasets, use 10+.

4. Technical Architecture & System Workflow (For Data Scientists)

The FAQ Mining System employs a modern, 13-stage NLP and Machine Learning pipeline designed for robust semantic extraction from unstructured conversational data.

The 13-Stage Pipeline

  1. Data Cleaning: Normalization, lowercasing, and removal of HTML tags and special characters.
  2. Language Detection: Detects Thai vs. English to route to appropriate processing functions.
  3. Tokenization & Stopword Removal: Utilizes pythainlp for Thai wrapping and standard NLTK for English.
  4. Lemmatization: Reduces words to their base forms to improve semantic matching.
  5. Semantic Embedding Extraction: Uses paraphrase-multilingual-MiniLM-L12-v2 (SentenceTransformers) to map text to 384-dimensional dense vectors.
  6. Deduplication (Exact & Fuzzy): Cosine similarity thresholding (configurable via UI) to merge identical intents.
  7. Dimensionality Reduction (UMAP): Reduces 384D to 5D locally preserving geometry for efficient clustering.
  8. Density-Based Clustering (HDBSCAN): Dynamically groups dense regions of questions into Topic Clusters, identifying outliers as "Noise (-1)".
  9. Centroid Calculation: Computes the geometric mean of embeddings within each cluster.
  10. Representative Selection: Selects the real question closest to the centroid as the single "Canonical Question".
  11. Answer Synthesis / Selection: Extracts the most relevant admin reply corresponding to the canonical question.
  12. Vector Search Indexing (FAISS): Builds an L2 Index using faiss-cpu for blazing-fast semantic search across the generated FAQ library.
  13. 3D Projection (PCA): Performs Singular Value Decomposition (SVD) to reduce embeddings to 3D space for interactive Three.js frontend rendering.

5. Deployment (Docker)

The system is fully containerized and production-ready. To deploy using Docker, ensure Docker Desktop is installed on your server.

Build & Run

# Step 1: Build the Docker image docker build -t faq-miner-ai . # Step 2: Run the container with volume mapping for persistence docker run -d -p 8000:8000 -v faq_data:/app/data --name faq_miner faq-miner-ai

The service will map to port 8000. Access the web interface at http://localhost:8000.

Process Management

The system is built with robust port management. If you start the pipeline manually using python backend/main.py --serve, it will automatically detect and kill any existing Python processes occupying port 8000, ensuring a clean single instance runs every time.
