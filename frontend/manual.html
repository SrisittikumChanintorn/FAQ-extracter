FAQ Mining

FAQ Mining is a local, offline-first system for extracting FAQ candidates from support datasets and organizing them into topic groups. It is designed for professional review workflows and is Thai-first (optimized for Thai customer support), while still understanding English.

Ollama (Local LLM) BAAI/bge-m3 Embeddings Batch Extraction FAISS Semantic Search

1. Prerequisites

Before you begin, confirm the following requirements are met:

1

Python

python --version

Python 3.9 or later is required. If not installed, download from python.org/downloads.

2

RAM

Get-CimInstance Win32_ComputerSystem | Select-Object TotalPhysicalMemory

Divide the value (bytes) by 1073741824 to get GB. For the recommended model, 8 GB RAM or higher is strongly recommended.

2. Install Ollama

Ollama enables you to run large language models locally on your machine.

1

Install via winget (recommended)

winget install Ollama.Ollama

After installation, close and reopen PowerShell to ensure the PATH is refreshed.

Alternative: Download the installer from ollama.com.

2

Verify installation

ollama --version

If you see ollama version 0.x.x, installation succeeded.

If "not recognized" appears, close and reopen PowerShell and try again.

Ollama typically runs as a background service after installation. In most cases, you do not need to run ollama serve manually.

3. Choose and load a model

Recommended model

ModelSizeMin. RAMNotes
scb10x/llama3.1-typhoon2-8b-instruct 5.0 GB 8 GB+ Recommended
1

Pull the model

ollama pull scb10x/llama3.1-typhoon2-8b-instruct

Download time depends on your connection and disk speed.

2

Verify

ollama list

Confirm the model name appears. Use this exact name in configuration (Step 5).

4. Install Python dependencies

1

Open the project directory

cd path\to\context_extract
2

Install dependencies

pip install -r requirements.txt

The first run may download the BAAI/bge-m3 embedding model; it will be cached for future use.

5. Configure the model

Set the LLM model name to match the output of ollama list.

A

Environment variable (per session)

$env:LLM_MODEL = "scb10x/llama3.1-typhoon2-8b-instruct"
B

Project configuration: edit backend/config.py

TOPIC_NAMER_MODEL = "scb10x/llama3.1-typhoon2-8b-instruct"

Use the exact name shown by ollama list.

6. Run the pipeline

1

Start the server

python backend/main.py --input data/conversations.json --serve

The console will display each stage. LLM extraction (Stage 4) is typically the longest step, especially on CPU.

2

Open the dashboard

http://localhost:8000

When the pipeline finishes, the server is ready and the dashboard will load.

8. Pipeline overview

1–3
Load → Clean → Filter

Load file, clean text, filter valid Q&A pairs.

4
LLM FAQ extraction (Ollama)

Data is split into batches; each batch is sent to the LLM to extract FAQ pairs and assign a group name. Results are merged across batches.

5
Merge batches

Groups are merged by name (embedding similarity); duplicate questions are deduplicated and mention counts summed.

6–7
Search index & save

Build FAISS index from group questions (BAAI/bge-m3). Save FAQs and analytics.

9. Output format

Results are written to data/faqs.json and exposed via GET /groups.

{
  "total_groups": 5,
  "groups": [
    {
      "group_id": 0,
      "group_name": "ปัญหาการเข้าสู่ระบบ",
      "total_faqs": 8,
      "faqs": [
        { "question": "ลืมรหัสผ่านต้องทำอย่างไร", "answer": "กรุณากด “ลืมรหัสผ่าน” ที่หน้าล็อกอินและทำตามขั้นตอน...", "mention_count": 3 }
      ]
    }
  ]
}

Use groups[].faqs to feed a chatbot knowledge base or export via the UI.

10. Configuration (config.py)

LLM (Ollama)

SettingDescription
TOPIC_NAMER_MODELMust match ollama list. Default: scb10x/llama3.1-typhoon2-8b-instruct.
OLLAMA_TIMEOUTEnv var. Default none (no timeout, suitable for CPU). Set seconds to limit.

Batch & merge

SettingDescription
FAQ_N_SPLITSNumber of data splits for parallel-style processing.
FAQ_EXTRACTION_BATCH_SIZEConversations per LLM call.
MERGE_QUESTION_SIMILARITY_THRESHOLDCosine threshold for merging duplicate questions (e.g. 0.88).
MERGE_GROUP_USE_EMBEDDINGUse embedding similarity for group names (recommended).

11. Troubleshooting

IssueSolution
ollama: not recognizedClose and reopen PowerShell after installing Ollama.
No FAQ pairs extractedEnsure Ollama is running and the model is pulled. Run ollama list and set TOPIC_NAMER_MODEL to that exact name.
Pipeline very slowNormal on CPU. No timeout by default. You can increase FAQ_EXTRACTION_BATCH_SIZE to reduce the number of LLM calls.
bge-m3 download slowFirst run only (~2.2 GB); subsequent runs use the cache.
Port 8000 in useThe pipeline may kill the previous process on start, or use --port 8001.
